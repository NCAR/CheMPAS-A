## Chemistry (expected new)

The upstream v8.3.1 has no chemistry coupling ŌĆö the `src/core_atmosphere/chemistry/`
path does not exist in the baseline. CheMPAS adds this directory entirely from scratch.
It contains 5 top-level `.F` source files and a `musica/` subdirectory with 2 `.F`
files, for 7 chemistry source files in total. The `chemistry/` path also includes a
`Makefile` and build artifacts (`.mod`, `.o`, `libchem.a`).

The top-level chemistry files implement: the generic chemistry interface
(`mpas_atm_chemistry.F`), the lightning-NOx source parameterization
(`mpas_lightning_nox.F`), the solar geometry calculation (`mpas_solar_geometry.F`),
and the TUV-x photolysis driver (`mpas_tuvx.F`). The `musica/` subdirectory contains
the MUSICA/MICM coupler (`mpas_musica.F`) and its build file.

The chemistry module is conditionally compiled via the `DO_CHEMISTRY` preprocessor
flag (set by the `MUSICA=true` build option). The coupling entry points ŌĆö init, step,
and finalize ŌĆö are called from `mpas_atm_core.F` and the runtime tracer injection
function `atm_extend_scalars_for_chemistry` is called from `mpas_atm_core_interface.F`.
A companion file `MPAS_MICM_tracer_consistency.md` (in `src/core_atmosphere/`) documents
the tracer naming conventions between MPAS and MICM.

### Build System

**Top-level `Makefile`** (+84/-13 lines): Three major additions:

1. **`llvm` target** ŌĆö new build target for macOS/LLVM/flang. Sets `OMPI_FC=flang`,
   `OMPI_CC=clang`, `OMPI_CXX=clang++` and passes `NOMPIMOD` (so `include 'mpif.h'`
   is used instead of `use mpi`). Exports OpenMPI wrapper variables to the shell.

2. **MUSICA support block** ŌĆö `ifeq "$(MUSICA)" "true"` block that calls
   `pkg-config --cflags musica-fortran` and `pkg-config --libs musica-fortran`
   at Makefile parse time, sets `-DMPAS_USE_MUSICA`, and re-appends NetCDF libraries
   after `-lmusica` to satisfy link-order requirements for TUV-x's embedded NetCDF objects.

3. **`musica_fortran_test` target** ŌĆö pre-link smoke test that compiles and links a
   minimal Fortran program against the MUSICA library; build fails fast with a helpful
   error if the library is not functional.

4. **`NETCDFF` variable support** ŌĆö separate variable for a Fortran-only NetCDF
   installation (needed on macOS where Homebrew's gfortran-built `libnetcdff` is
   ABI-incompatible with flang when MUSICA is enabled).

**`src/core_atmosphere/Makefile`** (+22/-10 lines): Added `CHEMISTRY` and `CHEMCORE`
variables and a `chemcore` target. When `CHEMISTRY=-DDO_CHEMISTRY`, the build recurses
into `chemistry/` to build `libchem.a`, explodes it into `libchem/`, and links its
`.o` files into `libdycore.a`. The compiler command lines for `.F` sources now include
`-I./chemistry` so that MUSICA `.mod` files are visible.

**`cmake/Modules/FindPnetCDF.cmake`** (+6/-4 lines): Minor fix ŌĆö 10 lines changed,
likely a path or variable name correction in the PnetCDF CMake finder.

**`src/framework/CMakeLists.txt`** (+3 lines): Minor addition.

## Notable Findings

- **Halo interface deduplication**: The `abstract interface` block for
  `halo_exchange_routine` was duplicated verbatim in three files in upstream v8.3.1
  (`mpas_atm_halos.F`, `mpas_atm_time_integration.F`, `mpas_atmphys_todynamics.F`).
  CheMPAS replaced all three with a single `#include "mpas_halo_interface.inc"`.
  This new `.inc` file is present in CheMPAS's `src/framework/` but absent from
  upstream ŌĆö it is a CheMPAS-originated refactor.

- **Chemistry tracers removed from Registry.xml**: MICM species (NO, NO2, O3, etc.)
  are not declared in `Registry.xml` at all. They are discovered at runtime by scanning
  the MICM YAML config and dynamically extending the `scalars` pool. This is a
  fundamental architectural departure from the upstream approach of build-time-declared
  tracers.

- **`physics_noaa/UGWP/` is entirely new and not in upstream**: The 7-file CIRES UGWPv1
  implementation under `src/core_atmosphere/physics/physics_noaa/UGWP/` has no upstream
  counterpart in v8.3.1. It is unclear whether this came from an intermediate ACOM-dev
  fork commit or was added independently in CheMPAS.

- **`mpas_pool_routines.F` null-pointer guards**: The 10-line addition to
  `mpas_pool_routines.F` adds `nullify()` calls before every pool lookup return path.
  This is a correctness fix: without these, callers that check `associated(ptr)` after
  a failed lookup may see stale non-null pointers from prior use of the same variable.

- **`mpas_dmpar.F` intent change**: The `inlist` parameter in an MPI scatter helper
  was changed from `pointer` to `intent(in)`. This is a bug fix ŌĆö passing a pointer
  where `intent(in)` is expected causes undefined behavior under strict compilers.

- **`mpas_stream_manager.F` error-handling hardening**: The 25-line addition adds
  early-return error checks after `MPAS_stream_list_remove` calls in alarm management.
  In upstream, list-remove failures were logged but the function continued executing
  on a corrupted state.
